## 构建步骤

### 方法1: 使用Android Studio构建（推荐）

1. **打开项目**
   - 打开 Android Studio
   - 选择 "Open" → 选择 `d:\game\tongxing\android` 目录

2. **同步Gradle**
   - Android Studio 会自动同步 Gradle
   - 如果遇到网络问题，请确保网络连接正常ాలుాలుాలుాలుాలుాలుాలుాలుాలుాలు

3. **构建APK**
   - 菜单栏: Build → Build Bundle(s) / APK(s) → Build APK(s)
   - 或者: Build → Generate Signed Bundle / APK → APK

4. **签名配置**
   - 使用现有的签名文件: `tongxing-release.keystore`
   - 别名: `tongxing`
   - 密码: `tongxing123`
   - 密钥密码: `tongxing123`

### 方法2: 命令行构建（需要网络连接）

1. **清理构建缓存**
```bash
cd d:\game\tongxing\android
.\gradlew.bat clean
```

2. **构建Release版本**
```bash
.\gradlew.bat assembleRelease
```

3. **如果遇到网络超时**
```bash
# 增加超时时间
set GRADLE_OPTS="-Dorg.gradle.internal.http.socketTimeout=60000 -Dorg.gradle.internal.http.connectionTimeout=60000"
.\gradlew.bat assembleRelease
```

### 方法3: 使用已下载的Gradle

如果网络连接有问题，可以：

1. **手动下载Gradle**
   - 下载地址: https://services.gradle.org/distributions/gradle-9.3.1-bin.zip
   - 保存到: `%USERPROFILE%\.gradle\wrapper\dists\gradle-9.3.1-bin\`

2. **设置本地Gradle路径**
   在 `gradle/wrapper/gradle-wrapper.properties` 中修改:
   ```
   distributionUrl=file:///C:/Users/用户名/.gradle/wrapper/dists/gradle-9.3.1-bin.zip
   ```

## 新功能 v0.1.6

### 导出系统修复
1. **权限修复**
   - 添加 `WRITE_EXTERNAL_STORAGE` 权限
   - 添加 `MANAGE_EXTERNAL_STORAGE` 权限 (Android 11+)

2. **三级导出策略**
   - **下载目录**: `内部存储/Download/同行导出/` (推荐)
   - **应用私有目录**: 最可靠，无需权限
   - **自定义目录**: 使用系统文件选择器

3. **错误处理增强**
   - 详细的错误分类和提示
   - 自动回退机制
   - 文件系统权限检查
   - 存储空间检查 (至少1MB)

### 使用方法
1. 在首页长按角色卡片
2. 选择"📤 导出数据"
3. 选择导出方式:
   - 保存到下载目录 (推荐)
   - 保存到应用私有目录
   - 选择保存位置 (系统文件选择器)

## 故障排除

### 网络连接问题
如果遇到gradle下载超时:
1. 检查网络连接
2. 尝试使用VPN
3. 手动下载gradle (方法3)
4. 使用Android Studio构建 (方法1)

### 构建失败
1. 清理构建缓存: `.\gradlew.bat clean`
2. 重新同步Gradle
3. 检查JDK版本 (需要JDK 17+)
4. 检查Android SDK版本

### 签名问题
如果签名失败:
1. 确认签名文件存在: `tongxing-release.keystore`
2. 检查密码: `tongxing123`
3. 检查别名: `tongxing`
